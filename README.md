# 问云派官网

## 项目介绍

问云派官网是一座“线上山门”，用于展示问云派愿景、立派金典、问云考核、问云名册、云灯留言、门派公告、问云雅集、联系山门、前台网易云歌单播放器、问云小院与后台管理。

网站核心气质是：清雅书院、线上山门、清醒温柔、同行自渡、真诚有界。当前版本已经重建公开官网、问云小院和执事后台的页面结构，核心登录、名册、云灯、考核、公告、雅集、邮件提醒和 Supabase 服务逻辑继续保留；问云小院偏私密任务院落，执事后台偏高密度运营工作台，并新增可直接编辑全量名册资料的名册控制台。

## 视觉与移动端设计说明

1. 当前页面按 `src/app/`、`src/features/`、`src/shared/` 三层组织：路由入口放在 `src/app/App.tsx`，公开官网、小院和后台分别放在 `src/features/public/`、`src/features/yard/`、`src/features/admin/`。
2. 新版视觉系统集中在 `src/shared/styles/taskbook.css`，使用深墨绿、朱红、湖蓝、米白和少量金色组成统一页面层级；公开官网、小院、后台共用基础组件，但通过不同布局和密度体现不同场景。
3. 首页保留“问云派”作为第一眼主信号，首屏使用真实山门图片做全幅背景，并新增云门罗盘、金线轨迹、入山路径和金典摘句，让首页更像一条可进入的山门动线。
4. 问云名册、问云考核、云灯留言和登录页统一使用新版互动页结构：桌面端是清晰的表单、列表、试卷和状态区，手机端自动变为单列；金典目录与考核题号在手机端改为横向吸顶滑动条，表单控件不低于 48px，按钮文字不竖排、内容不横向溢出。
5. 公告、雅集和联系页只展示真实服务层读取的数据或明确空状态，不再放静态假表单和容易误导的占位内容。
6. 问云小院采用深墨绿任务侧栏、浅色工作台、统计卡、资料卡、进度线、消息时间线和手机右侧滑出导航；执事后台采用更高信息密度的台账风格，并新增“名册控制台”，手机端可先横向滑动选择名帖，再进入单列编辑器直接编辑道名、江湖名、编号、序号、辈分、身份、状态、真实资料、联系方式、公开名册内容、管理员备注、归云堂状态和用户申请修改内容。
7. 当前新版公开页仍复用 `public/assets/guofeng-ui-20260509/` 中可用的背景图和页面插图，但导航、页脚、外层布局、色彩、按钮、表单、卡片和移动端规则已经独立重写。
8. 前台公开页新增网易云歌单音乐功能，页面右侧默认贴边显示带纸面背景的竖排歌词签，背景中只放当前歌词，不再展示拖动提示、歌单标题或其他说明；点击歌词签会从右往左展开网易云播放器控制台，长按歌词签可拖动，靠近左右侧边会自动贴边，真实歌词优先通过 `fetch-netease-lyrics` Edge Function 从网易云读取，失败时才回落到后台手写歌词。
9. 本次已按手机 390×844 真实浏览器视口重点检查首页、立派金典、问云考核、问云名册、登录页、小院和执事后台名册控制台；手机端导航、表单、题号、目录、名册编辑和前台音乐播放器均无横向溢出。

## 本次公开端重构审查记录

1. 路由逻辑：项目使用 `HashRouter`，公开页走 `PublicLayout`，登录后的小院走 `YardLayout`，管理员后台走 `AdminLayout`；当前路由入口位于 `src/app/App.tsx`。
2. 表单逻辑：云灯留言继续先校验内容，再调用 `submitCloudLantern` 写入待审核云灯；名册登记继续要求登录、最新问云考核合格、一人一帖，再调用 `submitJoinApplication`；问云考核继续按题库计分并调用 `submitWenxinQuizResult`。
3. 登录逻辑：登录页继续支持邮箱登录、旧编号登录、邮箱注册、找回密码和重置密码；旧编号仍通过 `normalizeLoginAccount` 转换为 Supabase 可识别账号，登录成功后仍补建 `profiles` 资料。
4. 数据读取逻辑：公告、雅集、联系设置、前台音乐设置和公开名册继续通过 `src/shared/services/` 统一读取，服务层仍保留 Supabase 未配置时的演示数据或空状态兜底。
5. 小院与后台逻辑：小院资料保存、邮箱绑定、密码修改、雅集报名、提醒已读、后台云灯审核、公告活动创建、SMTP 和归云堂设置仍集中在原服务层；执事后台名册控制台已接入 `updateApplicationDetails`，可直接保存完整名册资料。

## 环境要求

1. 操作系统：Windows 10 或 Windows 11。
2. Node.js：建议 `22.0.0` 或更高版本，本机已验证 `25.8.0` 可运行。
3. npm：建议 `10.0.0` 或更高版本，本机已验证 `11.11.0` 可运行。
4. 浏览器：最新版 Edge、Chrome 或其他现代浏览器。
5. 数据库：Supabase 项目一个，用于真实登录、问云小院、名册登记、云灯留言、公告、活动和后台管理。
6. Edge Function：`send-user-notice` 用于发送小院状态提醒邮件，`fetch-netease-lyrics` 用于服务端读取网易云真实歌词。

本项目依赖已在 `package.json` 中锁定版本：

| 依赖 | 版本 | 用途 |
| --- | --- | --- |
| `react` | `19.2.5` | 页面组件 |

### 五、配置 Supabase Edge Function

1. 安装 Supabase CLI。
2. 在 PowerShell 中登录 Supabase：

```powershell
supabase login
```

3. 关联你的 Supabase 项目：

```powershell
supabase link --project-ref 你的项目引用
```

4. 首次部署函数时可以设置 SMTP 密钥作为兜底配置。下面示例使用 QQ 邮箱 SMTP，请把占位内容换成你自己的邮箱和授权码，不要写进代码仓库：

```powershell
supabase secrets set SMTP_HOST=smtp.qq.com
supabase secrets set SMTP_PORT=465
supabase secrets set SMTP_USER=你的邮箱
supabase secrets set SMTP_PASS=你的邮箱授权码
supabase secrets set SMTP_FROM=你的邮箱
```

5. 部署小院邮件提醒 Edge Function：

```powershell
supabase functions deploy send-user-notice
```

6. 部署网易云真实歌词 Edge Function：

```powershell
supabase functions deploy fetch-netease-lyrics
```

注意：`supabase/config.toml` 已为 `fetch-netease-lyrics` 单独设置 `verify_jwt = false`。这个函数只读取公开网易云歌词，不写入数据库；关闭平台级登录校验后，浏览器的 `OPTIONS` 预检可以正常进入函数并拿到跨域响应头，避免真实歌词在前台被 CORS 拦截。

7. 邮件提醒触发位置：

```text

管理员审核云灯状态
管理员调整活动报名状态
用户报名或取消雅集
```

注意：邮件发送失败不会影响审核或报名，失败原因会写入问云小院的消息提醒里。真实歌词函数不需要 SMTP 密钥，只负责服务端读取网易云公开歌词接口，避免浏览器跨域限制。

也可以不使用 Supabase Secrets，在管理后台设置 SMTP 服务：

1. 登录掌门或执事账号。
2. 打开 `/admin/settings`。
3. 在“SMTP 邮件服务”里填写 SMTP 主机、端口、账号、授权码、发件人。
4. 勾选“启用后台 SMTP 配置”。
5. 保存后，`send-user-notice` 会优先使用后台 SMTP 设置；后台未启用时才使用 Supabase Secrets。

### 六、初始化管理员

1. 打开 `/login` 页面。
2. 使用邮箱和密码注册一个账号。
3. 回到 Supabase SQL 编辑器，把下面语句里的邮箱改成你的管理员邮箱：

```sql
update public.profiles
set role = 'founder'
where id = (select id from auth.users where email = '你的邮箱@example.com');
```

4. 回到官网 `/login` 登录。
5. 登录成功后所有账号都会进入 `/yard` 问云小院；掌门或执事会在小院桌面侧栏、手机顶部身份条和右侧滑出导航里看到“前往执事后台”入口，也可以直接打开 `/admin`。

如果你之前已经开启邮箱确认并注册过账号，导致登录仍被拦住，可以执行项目里的 `supabase/admin_confirm_founder.sql` 手动确认管理员邮箱并提权。

角色说明：

| 角色 | 含义 |
| --- | --- |
| `founder` | 掌门，拥有后台管理权限 |
| `admin` | 执事，拥有后台管理权限 |
| `member` | 普通同门，可进入问云小院，不能进入管理后台 |

### 七、部署到 GitHub Pages

1. 把代码推送到 GitHub 的 `main` 分支。
2. 打开仓库设置里的 Pages 功能。
3. 部署来源选择 GitHub Actions。
4. 打开仓库的 `Settings` → `Secrets and variables` → `Actions`，在仓库密钥中添加：

```text
VITE_SUPABASE_URL
VITE_SUPABASE_ANON_KEY
```

也兼容下面这组简短名称，但优先推荐上面的 `VITE_` 名称：

```text
SUPABASE_URL
SUPABASE_ANON_KEY
```

注意：电脑里的 `.env.local` 只对本地运行生效，不会自动上传到 GitHub。部署到 GitHub Pages 时，必须在 GitHub 仓库里单独添加上面的仓库密钥或仓库变量。

5. 推送代码后，`.github/workflows/deploy.yml` 会自动生成 `public/env.js` 线上配置，再构建并发布 `dist` 静态网站。
6. 如果刚添加密钥后页面仍提示未配置，请到 GitHub 的 `Actions` 页面重新运行最新一次部署。

部署后页面地址会使用 `/#/` 形式，例如：

```text
https://你的用户名.github.io/仓库名/#/
https://你的用户名.github.io/仓库名/#/canon
```

这样做是为了兼容 GitHub Pages 静态托管，避免刷新页面或打开子页面时误入 404。

## 使用教程

### 访问官网首页

打开 `/` 可以看到问云派插画山门首屏、核心功能插画卡片、门派精神、名册流程、云灯预览、公告入口和右侧贴边的前台歌词签。首屏底部的“立派金典、问云考核、云灯留言、问云小院、公告活动”五个入口已居中排列，点击后会跳到对应页面。

### 使用前台音乐播放器

1. 管理员在后台开启前台音乐后，公开官网页面右侧会默认贴边显示竖排歌词签。
2. 歌词签带纸面背景，但背景里只显示当前歌词文字本身，没有底部听歌条、歌单标题、拖动提示或其他说明。
3. 点击歌词签，会从右往左展开网易云播放器控制台，可直接播放后台设置的歌单。
4. 长按歌词签可以拖动到任意位置；如果拖到左右侧边附近，松手后会自动贴边，方便手机单手使用。
5. 如果后台开启自动播放但浏览器拦截，页面会出现全屏提示；点击提示层任意位置后会直接开始播放音乐，不会先弹出播放器控制台。
6. 如果已经部署 `fetch-netease-lyrics` Edge Function，前台会优先读取网易云歌单第一首歌的真实歌词；读取失败时才使用后台填写的歌词轮播文本。
7. 如果后台关闭播放器，或没有填写可识别的歌单编号，前台不会显示歌词签。

### 切换页面

在任意页面向下阅读后，再点击顶部导航、手机菜单、小院侧栏或后台侧栏切换页面，新页面都会自动回到顶部，避免沿用上一个页面的滚动位置。

### 阅读立派金典

打开 `/canon` 可以阅读完整《问云派立派金典》正式初稿，内容包含卷首语、十九章正文和微信群公告简版。桌面端左侧有章节目录，手机端章节目录会变成横向吸顶滑动条，可边读边切换章节。

### 使用问云名册

1. 打开 `/join`。
2. 页面上半部分先随机展示几位已经入册通过或已联系的同门，只露出道名、江湖名、身份、公开地区和宣言摘要。
3. 点击任意名册卡片会打开公开详情弹窗，可查看编号、身份、公开地区、入册时间、入派宣言、兴趣标签和同行期待；真实姓名、联系方式和出生年份不会公开。
4. 名册数量较多时可点击“换一组”重新随机展示几位；手机端名册卡片可横向滑动查看。
5. 页面下半部分是“递交名帖”纸卡；未登录、未通过问云考核或已经递交过名帖时，会显示对应提示和入口。
6. 点击“问云小院”登录或注册账号。
7. 先打开 `/wenxin-quiz` 完成问云考核。最新一次成绩达到 80 分以上后，名册登记入口才允许提交。
8. 回到登记入口填写道名、江湖名、真实姓名、联系方式、出生年份、所在城市、性别、宣言、兴趣爱好和同行期待等内容。
9. 身份默认是“同门”，用户不能自行更改；如需设置掌门、执灯长老、云纪执事、文案执事或雅集执事，需要管理员在后台调整。
10. 江湖名、所在城市、宣言、兴趣爱好和同行期待会进入公开名册；真实姓名、联系方式和出生年份只给本人和后台管理员核对。
11. 道名必须以“云”字开头，长度为 2 到 3 个字，例如“云初”“云灯”。
12. 勾选认同问云派金典。
13. 点击“提交名册登记”。
14. 登记会进入后台待审核列表，系统会默认生成类似“问云-云-001”的编号。
15. 每个账号只能递交一份名帖；如果已经递交过，页面会指引你去问云小院的“我的资料”修改资料，或到“我的名帖”查看审核状态。
16. 审核状态会回到你的问云小院消息提醒里。
17. 旧名册导入后，同门可用三位短编号登录；有出生年份时密码为年份，没有年份时密码为短编号本身。

注意：当前名册资料以“道名、江湖名、真实姓名、联系方式、出生年份、所在城市、性别、宣言、兴趣爱好、同行期待、身份、编号、状态”为准；旧的公开故事、后台故事原文、线下雅集意愿、普通备注、封面和羁绊状态已经不再使用，执行最新字段统一脚本后会从数据库中删除。用户中心显示的道名会跟随名册资料同步。

### 参加问云考核

1. 打开 `/wenxin-quiz`。
2. 未登录时会先显示“登录后开考”纸卡，登录问云小院后才会展示完整试卷并保存成绩。
3. 考核共 30 题：单选题 25 题，每题 3 分；多选题 5 题，每题 5 分，总分 100 分。
4. 顶部会显示答题进度、剩余题数和当前得分；右侧会显示考核须知、本次状态和题号导航。
5. 页面一次只显示一道题，选项会以试卷卡展示，选中后会变成深青色；手机端会自动改成单列大按钮，方便拇指点击。
6. 可以点击题号导航跳题；手机端题号导航会固定在当前题卡上方并横向滑动，不会撑开页面。
7. 到最后一题且全部题目都已作答后，点击“交卷”。
8. 90 到 100 分表示深知金典，可登记入册。
9. 80 到 89 分表示基本明愿知界，可登记入册。
10. 70 到 79 分建议云纪执事说明金典重点后，再决定是否入册。
11. 70 分以下建议暂缓入册，重读金典后再考。
12. 系统会保存每次考核记录，小院和后台展示最新一次结果，名册登记也只看最新一次是否合格。

### 点亮云灯

1. 打开 `/cloud-lantern`。
2. 先登录问云小院。
3. 在左侧“点一盏云灯”纸卡里填写署名、心情标签和留言内容。
4. 可选择匿名展示；开启匿名后署名输入框会停用。
5. 提交后默认进入待审核状态。
6. 管理员审核通过后，留言会在右侧“云灯墙”公开展示。
7. 审核状态会回到你的问云小院消息提醒里。

### 进入问云小院

1. 打开 `/login`。
2. 页面左侧是问云小院山门水墨图，右侧是登录纸卡。
3. 已有账号可选择“邮箱登录”或“旧编号登录”；旧编号可填写 001 这类短编号。
4. 新用户点击“立即注册”，使用邮箱和密码注册账号。
5. 注册或登录成功后会进入 `/yard` 问云小院，管理员账号也会保留自己的小院。
6. 问云小院包含小院总览、我的资料、我的名帖、我的云灯、我的雅集和消息提醒；桌面端使用左侧深青任务导航，手机端使用顶部身份条和右下角圆形导航按钮，点击后菜单会从右向左展开并悬浮在页面之上，方便单手切换页面。
7. 本地只想预览 UI 时，可打开 `/?demo-yard=1#/yard` 使用演示资料查看小院总览，不会写入数据库。
8. 小院总览会用统计卡展示任务进度、名帖、云灯和未读提醒，并在下方展示本册进度、下一步可做和最近提醒。
9. 如果名帖已经“已通过”或“已联系”，且后台已经上传归云堂二维码，小院总览会全屏弹出入群二维码。弹窗提供“我已入群”和“暂时关闭”两个按钮。
10. 点击“我已入群”后，会同步更新本人名帖的“已入归云堂”状态，之后二维码不再主动弹出，但小院总览会保留“归云堂入群”卡片，可随时再次打开二维码。
11. 点击“暂时关闭”只会关闭本次弹窗，不会修改名册状态；下次重新进入小院时，若仍未确认入群，系统还会再次提醒。
12. 掌门或执事账号会在小院侧栏、手机顶部身份条和右侧滑出菜单里看到执事后台入口，普通同门不会看到。
13. 我的资料分为账号安全、小院名片、名册资料和小院展示资料：账号安全可绑定或更换真实邮箱，也可以修改登录密码。
14. 名册资料里，真实姓名、出生年份和身份会以灰色只读方式展示，用户不能自行更改。
15. 江湖名、性别、所在城市、宣言、兴趣爱好和同行期待可以直接保存。
16. 道名和联系方式可以提交修改申请，管理员审核同意后才会生效。
17. 我的名帖会展示最新审核状态、横向审核进度和完整名帖详情。
18. 我的云灯会展示全部、审核中、已公开和未通过数量，并按列表展示每盏云灯的内容、心情、署名和审核时间。
19. 我的雅集会展示活动封面、时间、地点、人数、报名备注，并可报名或取消报名。
20. 消息提醒会展示站内通知、业务类型、邮件发送状态和已读按钮。
21. 旧编号账号绑定邮箱后，可用真实邮箱和当前密码登录；修改密码后，下次登录使用新密码。
22. 忘记密码时，在 `/login` 点击“忘记密码，用绑定邮箱找回”，填写已经绑定的真实邮箱，收到邮件后回到网站设置新密码。
23. 如果旧编号账号还没有绑定真实邮箱，就不能通过邮箱找回密码；可请超级管理员在执事管理里重置密码。

### 报名问云雅集

1. 打开 `/events` 或 `/yard/events`。
2. 先登录问云小院。
3. 找到想参加的活动，填写报名备注。
4. 点击“报名雅集”。
5. 报名后可在问云小院查看状态，也可以取消报名。

### 管理后台

后台地址为 `/admin`，必须先通过 `/login` 登录管理员账号。普通用户访问 `/admin` 会自动回到 `/yard`，管理员也可以从问云小院一键进入后台。

后台包含：

1. 后台总览：查看待审核名帖、待审核云灯、公告总数和活动总数，并快速看到最新待办。
2. 名册控制台：左侧为名册索引，可按道名、江湖名、真实姓名、编号、联系方式和地区搜索，也可按待审核、已通过、已联系、未通过、暂存、已退派筛选；右侧为全量编辑器，可直接编辑道名、江湖名、名册编号、入册序号、辈分字、门派身份、状态、入册时间、真实姓名、微信号、备用联系、出生年份、性别、城市、公开地区、宣言、兴趣爱好、同行期待、申请理由、管理员备注、归云堂状态和用户申请修改内容；管理员也可彻底删除名帖，删除后不可恢复。
3. 云灯审核：按作者、内容和心情搜索留言，按状态筛选，并在台账中通过或拒绝留言。
4. 公告管理：左侧创建草稿或发布公告，右侧查看公告列表、分类、发布状态和更新时间。
5. 活动管理：左侧创建线上或线下雅集，右侧查看活动台账，下方查看报名名单并调整报名状态。
6. 执事管理：超级管理员可搜索成员，将成员设置为执事或撤回普通成员；也可以在账号抽屉中修改普通成员或执事的绑定邮箱，并在需要时重置密码。密码不会在页面回显，超级管理员账号本身请到问云小院修改。
7. 站点设置：三栏维护联系山门说明、前台网易云歌单播放器、归云堂入群二维码图片上传与预览、SMTP 邮件服务设置。

### 配置前台音乐播放器

1. 登录掌门或执事账号。
2. 打开 `/admin/settings`。
3. 在“前台音乐”区域勾选“启用前台播放器”。
4. 在“网易云歌单链接”里粘贴歌单地址，例如 `https://music.163.com/#/playlist?id=歌单编号`，系统会自动提取歌单编号。
5. 如果链接无法识别，可以在“歌单编号”里手动填写纯数字编号。
6. 填写播放器标题，点击右侧歌词签展开的播放器控制台会显示这个标题。
7. 在“歌词轮播文本”里按行填写歌词，每行一句；如果真实歌词函数未部署、网易云接口失败或当前歌曲没有歌词，前台会使用这里的文本兜底。
8. 如需请求自动播放，可勾选“请求自动播放”；实际是否自动播放仍由用户浏览器决定。
9. 点击“保存音乐设置”后，公开官网会读取最新歌单设置。

### 配置归云堂入群二维码

1. 先执行 `supabase/migrations/20260429173000_guiyuntang_qr_settings.sql`。
2. 登录掌门或执事账号。
3. 打开 `/admin/settings`。
4. 在“归云堂入群二维码”区域点击文件上传框，选择二维码图片。建议使用清晰方形图，文件不超过 2MB。
5. 上传后先查看页面里的二维码预览，确认无误后填写审核通过提示文案和保密提醒。
6. 点击“保存归云堂二维码设置”，图片才会正式写入受保护的后台设置表。
7. 打开 `/admin/applications`，把名帖状态改为“已通过”或“已联系”。
8. 成员进入 `/yard` 时会全屏弹出归云堂入群二维码；弹窗里有“我已入群”和“暂时关闭”两个按钮。
9. 成员点击“我已入群”后，会写入独立入群字段，弹窗不再自动出现，但用户仍可在小院总览的“归云堂入群”卡片里再次查看二维码。
10. 成员点击“暂时关闭”不会修改名册状态，只是本次先不处理。
11. 如果成员已经线下确认入群，管理员也可以在名帖详情里勾选“已入归云堂”，这不会改变名帖审核状态。

注意：归云堂二维码严禁向外部透露。本项目不会把二维码放进 `public` 静态目录，也不会写进 README 或 SQL；二维码只保存在受 Supabase RLS 保护的设置表中，只有管理员和名帖已通过、已联系的本人可读取。

### 全站视觉说明

1. 前台官网采用“现代东方线上山门”视觉方向，首页首屏第一眼展示问云派，并用云门罗盘、金线轨迹、入山路径和金典摘句建立更强的记忆点。

## 常见问题排查

### 页面显示演示数据

原因：没有配置 `.env.local`，或 Supabase 地址、公开匿名密钥不正确。

解决方法：

1. 检查 `.env.local` 是否在项目根目录。
2. 检查 `VITE_SUPABASE_URL` 是否以 `https://` 开头。
3. 检查 `VITE_SUPABASE_ANON_KEY` 是否为 Supabase 的公开匿名密钥。
4. 修改后重新运行 `npm run dev`。

### GitHub Pages 部署后仍显示未配置 Supabase

原因：本地 `.env.local` 不会跟随代码部署，GitHub Actions 没有读取到仓库密钥或仓库变量。

解决方法：

1. 打开 GitHub 仓库的 `Settings` → `Secrets and variables` → `Actions`。
2. 在 `Repository secrets` 中添加 `VITE_SUPABASE_URL` 和 `VITE_SUPABASE_ANON_KEY`。
3. 如果你已经用了 `SUPABASE_URL` 和 `SUPABASE_ANON_KEY`，当前工作流也会兼容读取。
4. 回到 `Actions` 页面，重新运行最新部署。
5. 部署日志里看到“已写入线上 Supabase 公开配置。”就表示线上已拿到配置。
6. 浏览器强制刷新页面，地址建议使用 `https://你的用户名.github.io/仓库名/#/`。

### 后台提示没有权限

原因：账号已登录，但 `profiles.role` 不是 `admin` 或 `founder`。

解决方法：在 Supabase SQL 编辑器执行管理员初始化语句，把对应邮箱改为 `founder`。

### 登录提示邮箱还没有确认

原因：Supabase 开启了邮箱确认。本项目要求邮箱和密码直接注册登录，不走确认邮件流程，所以需要关闭邮箱确认。

解决方法：

1. 打开 Supabase 后台。
2. 进入 `Authentication` → `Sign In / Providers`。
3. 关闭邮箱确认。
4. 同时关闭安全邮箱变更，方便旧编号账号后续绑定真实邮箱。
5. 回到 `/login` 重新注册或登录。
6. 如果旧账号已经被确认流程卡住，可以在 Supabase SQL 编辑器执行：

```sql
update auth.users
set email_confirmed_at = now()
where email = '你的邮箱@example.com'
  and email_confirmed_at is null;
```

7. 管理员账号还需要检查 `public.profiles` 表里该用户的 `role` 是否为 `founder` 或 `admin`。

### 绑定邮箱后没有立即生效

可能原因：Supabase 仍然开启了邮箱确认或安全邮箱变更，系统需要确认新邮箱，甚至会要求确认旧邮箱。旧编号账号的旧邮箱是内部邮箱，用户无法收到旧邮箱确认邮件。也可能是没有执行邮箱直绑函数脚本。

解决方法：

1. 打开 Supabase 后台。
2. 进入 `Authentication` → `Sign In / Providers`。
3. 关闭邮箱确认。
4. 关闭安全邮箱变更。
5. 执行 `supabase/migrations/20260428230500_bind_legacy_account_email.sql`。
6. 回到 `/yard/profile` 重新绑定真实邮箱。

### 找回密码邮件收不到或链接打不开

可能原因：

1. 账号还没有绑定真实邮箱，仍然是 `001@wenyun.local` 这类内部邮箱。
2. Supabase 的 Auth 邮件服务被频率限制。
3. 邮件进了垃圾箱。
4. `Authentication` → `URL Configuration` 里没有添加 `?mode=reset-password` 回跳地址。
5. 线上域名和 Supabase 允许回跳地址不一致。

解决方法：

1. 先确认该成员已经在问云小院绑定真实邮箱，或由超级管理员在执事管理里直接修改绑定邮箱。
2. 在 `/login` 点击“忘记密码，用绑定邮箱找回”，只填写真实邮箱，不填写旧编号。
3. 检查邮箱垃圾箱和拦截箱。
4. 在 Supabase 后台的 `Redirect URLs` 中添加线上地址和重置地址，例如 `https://你的域名/*`、`https://你的域名/?mode=reset-password`。
5. 本地测试时添加 `http://localhost:5176/*` 和 `http://localhost:5176/?mode=reset-password`。
6. 如果用户没有绑定邮箱或邮件长期收不到，可由超级管理员进入“执事管理”直接重置密码。

### 执事管理修改邮箱或密码提示函数错误

报错内容通常包含 `gen_salt` 找不到，或 `column reference "user_id" is ambiguous`。

原因：部分 Supabase 项目把加密扩展函数放在 `extensions` 命名空间，旧版账号维护函数没有把这个命名空间加入搜索路径；另外函数返回列里也有 `user_id`，旧版 SQL 没有给 `auth.identities.user_id` 加表名前缀，数据库会分不清字段来源。

解决方法：

1. 执行 `supabase/fix_steward_account_password_hash.sql`。
2. 刷新 `/admin/stewards`。
3. 重新展开成员卡片的“账号设置”，再次保存邮箱或密码。
4. 如果只改邮箱不改密码，也可以把密码输入框留空。

### 3199912548@qq.com 需要绑定 001 编号

原因：`3199912548@qq.com` 已经是独立管理员账号，不能再直接把 001 内部账号改成这个邮箱，否则会出现“邮箱已注册”的冲突。

解决方法：

1. 先确认旧名册已导入，数据库里存在 `问云-云-001`。
2. 执行 `supabase/migrations/20260428231500_steward_management.sql`，把 `3199912548@qq.com` 设为超级管理员。
3. 执行 `supabase/bind_admin_3199912548_to_001.sql`，把 001 名册归属转移到这个管理员账号。
4. 重新登录 `3199912548@qq.com`，进入问云小院和管理后台。

### 执事管理页面提示权限不足

可能原因：当前账号不是 `founder` 超级管理员，或者浏览器还停留在旧登录会话里，或者没有执行执事管理 SQL。

解决方法：

1. 执行 `supabase/migrations/20260428231500_steward_management.sql`。
2. 执行 `supabase/fix_3199912548_founder_access.sql`。
3. 确认执行结果里 `后台角色` 是 `founder`。
4. 在网站里退出后台。
5. 使用 `3199912548@qq.com` 或 `001` 重新登录后台。

### 执事管理提示函数返回类型不匹配

报错内容通常是 `structure of query does not match function result type`。

原因：不同 Supabase 项目里 `auth.users.email` 可能是 `varchar`，而执事管理函数返回表定义要求 `text`，PostgreSQL 要求返回类型完全一致。

解决方法：

1. 执行 `supabase/fix_steward_management_query_structure.sql`。
2. 执行完成后刷新 `/admin/stewards`。
3. 如果仍然提示权限不足，再执行 `supabase/fix_3199912548_founder_access.sql` 并重新登录。

### 执事管理修改邮箱或密码失败

可能原因：

1. 没有执行 `supabase/migrations/20260429085600_steward_account_management.sql`。
2. 当前登录账号不是 `founder` 超级管理员。
3. 正在修改超级管理员账号本身。
4. 新邮箱已经被另一个账号占用。
5. 新邮箱仍然是 `001@wenyun.local` 这类内部邮箱。

解决方法：

1. 先执行 `supabase/migrations/20260428231500_steward_management.sql`。
2. 再执行 `supabase/migrations/20260429085600_steward_account_management.sql`。
3. 退出网站并用超级管理员账号重新登录。
4. 如果提示邮箱已被占用，先在执事管理里找到占用该邮箱的账号，把它改成另一个真实邮箱，再给当前成员绑定目标邮箱。
5. 如果要修改超级管理员自己的邮箱或密码，请进入问云小院的“我的资料”操作。

### 问云小院进不去

可能原因：

1. 没有配置 Supabase。
2. 没有关闭邮箱确认。
3. 没有执行 `20260428133000_wenyun_yard_and_notifications.sql`。
4. `profiles` 表没有自动生成当前用户资料。

解决方法：

1. 确认 `.env.local` 或 GitHub Actions 密钥已经配置。
2. 确认 Supabase 已关闭邮箱确认。
3. 执行 `supabase/migrations/20260428133000_wenyun_yard_and_notifications.sql`。
4. 重新打开 `/login` 注册并登录。

### 旧编号账号登录失败

可能原因：

1. 没有执行旧名册导入脚本。
2. 输入的是完整编号但末尾数字不对，例如编号不是 `001` 到 `029`。
3. 密码填错：有出生年份的同门密码是年份，没有年份的同门密码是三位编号。
4. 前面没有先清空测试名册，导致旧编号或名册数据冲突。
5. 已经执行过旧版导入脚本，但旧编号账号的 Auth 令牌字段还是空值，Supabase 会报 `Database error querying schema`。

解决方法：

1. 先执行 `supabase/clear_roster_lantern_test_data.sql`。
2. 再执行 `supabase/migrations/20260428173000_import_yunqi_legacy_roster.sql`。
3. 再执行 `supabase/migrations/20260429084000_roster_cards_profile_permissions.sql` 补齐出生年份和最新名册权限。
4. 如果之前已经导入过旧名册，直接执行 `supabase/fix_legacy_roster_auth_accounts.sql` 修复旧编号账号。
5. 登录时账号填写短编号，例如 `001`。
6. 密码优先填写出生年份；如果该同门没有年份，就填写短编号本身，例如 `022`。

### 小院邮件提醒发送失败

可能原因：

1. 没有部署 `send-user-notice` Edge Function。
2. 没有执行 `20260428144500_admin_smtp_settings.sql`。
3. 后台 SMTP 设置未启用，且没有配置 Supabase SMTP 密钥。
4. SMTP 授权码错误。
5. 邮箱服务商限制登录或发信。

解决方法：

1. 执行 `supabase functions deploy send-user-notice`。
2. 执行 `supabase/migrations/20260428144500_admin_smtp_settings.sql`。
3. 登录 `/admin/settings`，在“SMTP 邮件服务”里填写配置并启用。
4. 如果不想用后台配置，也可以在 Supabase Secrets 中配置 `SMTP_HOST`、`SMTP_PORT`、`SMTP_USER`、`SMTP_PASS`、`SMTP_FROM`。
5. QQ 邮箱通常使用 `SMTP_HOST=smtp.qq.com` 和 `SMTP_PORT=465`。
6. 不要把 SMTP 授权码写进前端代码、README 或 Git 仓库。
7. 到问云小院消息提醒里查看 `邮件状态` 和失败原因。

### 前台没有显示真实歌词

可能原因：

1. 没有执行 `20260516131000_music_player_settings.sql`，数据库里没有 `music_player` 设置记录。
2. 没有带着 `supabase/config.toml` 重新部署 `fetch-netease-lyrics` Edge Function，浏览器预检请求仍可能被 CORS 拦截。
3. 没有部署 `fetch-netease-lyrics` Edge Function。
4. 后台填写的歌单编号不正确，或歌单里第一首歌没有歌词。
5. 网易云公开接口临时限制访问。
6. 本地没有配置 Supabase，只能使用演示歌词或后台填写的歌词。

解决方法：

1. 执行 `supabase/migrations/20260516131000_music_player_settings.sql`。
2. 确认 `supabase/config.toml` 里存在 `[functions.fetch-netease-lyrics]` 和 `verify_jwt = false`。
3. 执行 `supabase functions deploy fetch-netease-lyrics`。
4. 打开 `/admin/settings`，确认“前台音乐”里已经填写正确的网易云歌单链接或编号，并保存到数据库。
5. 在“歌词兜底文本”里填写兜底歌词，避免网易云接口失败时前台空白。
6. 重新打开公开官网，等待右侧歌词签出现；点击歌词签可打开播放器控制台。

### 名册登记提交失败

可能原因：

1. Supabase SQL 没有执行完整。
2. RLS 策略没有创建成功。
3. 表单没有勾选认同问云派金典。
4. 没有执行 `20260428110000_wenyun_roster.sql`，导致数据库缺少性别、身份、辈分字或编号字段。
5. 没有执行 `20260428125300_roster_names_and_review.sql`，导致数据库缺少江湖名、真实姓名、暂存或已退派字段规则。
6. 没有执行 `20260428133000_wenyun_yard_and_notifications.sql`，导致数据库缺少 `user_id` 或登录用户提交策略。
7. 没有执行 `20260429110500_wenxin_quiz_gate.sql`，导致数据库缺少问云考核结果表或登记门槛策略。
8. 没有先登录问云小院。
9. 最新一次问云考核没有达到 80 分。
10. 道名没有以“云”字开头，或长度不是 2 到 3 个字。

解决方法：

1. 先执行 `supabase/migrations/20260428090000_init_wenyunpai.sql`。
2. 再执行 `supabase/migrations/20260428110000_wenyun_roster.sql`。
3. 再执行 `supabase/migrations/20260428125300_roster_names_and_review.sql`。
4. 再执行 `supabase/migrations/20260428133000_wenyun_yard_and_notifications.sql`。
5. 最后执行 `supabase/migrations/20260429110500_wenxin_quiz_gate.sql`。
6. 确认 `join_applications` 表存在，并且有 `jianghu_name`、`real_name` 和 `user_id` 字段。
7. 确认 `roster_entries` 公开视图存在，并且公开字段里有 `jianghu_name`。
8. 确认已经登录问云小院。
9. 打开 `/wenxin-quiz` 完成问云考核，并确认最新成绩达到 80 分以上。
10. 确认登记表单已勾选金典确认，且道名符合“云”字开头的规则。

### 名册登记提示先完成问云考核

原因：问云名册已经和问云考核绑定，只有最新一次问云考核达到 80 分以上，才可以提交登记。

解决方法：

1. 先登录问云小院。
2. 打开 `/wenxin-quiz`。
3. 完成 25 道单选题和 5 道多选题。
4. 点击“交卷并记录成绩”。
5. 如果分数达到 80 分以上，再回到 `/join` 展开登记入口填写名帖。
6. 如果已经合格但仍然提示不能登记，请确认线上数据库已经执行 `supabase/migrations/20260429110500_wenxin_quiz_gate.sql`。

### 名册登记提示已经递交过名帖

原因：系统已经限制每个账号只能递交一份名帖，避免一个人重复提交多份审核记录。

解决方法：

1. 打开 `/yard/profile`，在问云小院里修改自己的资料。
2. 如果要修改道名或联系方式，提交后会进入管理员审核。
3. 打开 `/yard/applications` 可以查看自己的名帖状态和管理员备注。
4. 如果确实需要重新提交，请联系掌门或执事在后台彻底删除旧名帖后再递交。
5. 如果线上仍允许重复提交，请确认已经执行 `supabase/migrations/20260429164500_single_application_and_admin_delete.sql`。

### 后台彻底删除名帖失败

可能原因：

1. 当前账号不是掌门或执事。
2. 没有执行最新的一人一帖和删除权限脚本。
3. 浏览器还停留在旧登录会话里。

解决方法：

1. 执行 `supabase/migrations/20260429164500_single_application_and_admin_delete.sql`。
2. 退出网站后重新登录管理员账号。
3. 打开 `/admin/applications`，展开名帖详情，在底部点击“彻底删除”。
4. 删除后不可恢复；如果只是临时不公开，建议把状态改为“暂存”或“已退派”。

### 后台看不到归云堂二维码提示

可能原因：

1. 没有执行 `20260429173000_guiyuntang_qr_settings.sql`。
2. 没有在 `/admin/settings` 上传并保存二维码。
3. 名帖状态还不是“已通过”或“已联系”。
4. 当前账号不是掌门或执事，无法读取后台保密设置。

解决方法：

1. 执行 `supabase/migrations/20260429173000_guiyuntang_qr_settings.sql`。
2. 执行 `supabase/migrations/20260501113000_guiyuntang_joined_flag.sql`。
3. 进入 `/admin/settings` 上传归云堂二维码并保存。
4. 回到 `/admin/applications`，把名帖状态改为“已通过”或“已联系”，即可看到醒目入群提示。
5. 成员确认进群后，可由成员在小院点击“我已入群”，也可由管理员勾选“已入归云堂”；提示会收起，但可点击“查看二维码”。
6. 不要把二维码上传到 `public` 目录，也不要发到前台页面或公开文档。

### 用户中心道名和资料里的道名不一致

原因：旧版本同时在 `profiles.nickname` 和 `join_applications.nickname` 中保存道名，后来修改名册资料时可能只改到其中一处。

解决方法：

1. 执行 `supabase/migrations/20260501110000_unify_roster_profile_fields.sql`。
2. 执行 `supabase/check_roster_field_unification.sql` 检查结果。
3. 确认“废弃字段剩余数量”为 `0`。
4. 确认“道名不一致数量”为 `0`。
5. 退出网站并重新登录问云小院。

### 编号需要往前补齐

原因：待审核名帖提前生成了 `问云-云-030` 这类编号，或者中间编号被拿掉后，后面的 `031、032` 没有自动变成 `030、031`。

解决方法：

1. 执行 `supabase/fix_roster_number_gaps.sql`。
2. 查看脚本输出的“本次被释放编号的待审核名帖”，确认释放了哪些待审核编号。
3. 查看“本次编号往前补齐的名帖”，确认 `031 → 030`、`032 → 031` 这类变化已经发生。
4. 确认最后一张检查结果表中，“待审核仍占号数量”“入册序号不连续数量”“名册编号不连续数量”都为 `0`。
5. 暂存、已退派、未通过、已审核、已联系的已有编号都会保留在序列里，并随整体压号往前移动。
6. 以后待审核名帖不会提前分配公开编号；管理员把名帖改成已审核或已联系并保存后，系统才会在没有编号时分配新编号。

### 用户中心保存资料后刷新还是旧内容

可能原因：

1. 线上前端还停留在旧版本，保存名册资料时没有把参数传给最新数据库函数。
2. 没有执行 `supabase/migrations/20260501110000_unify_roster_profile_fields.sql`。
3. 修改的是道名或联系方式，这两项不会立即覆盖正式资料，而是先进入管理员审核。

解决方法：

1. 重新部署最新前端代码。
2. 执行 `supabase/migrations/20260501110000_unify_roster_profile_fields.sql`。
3. 用户在 `/yard/profile` 保存名册资料后刷新页面，江湖名、性别、所在城市、宣言、兴趣爱好和同行期待应保持新值。
4. 如果修改了道名或联系方式，管理员进入 `/admin/applications`，点击“资料修改”筛选，展开对应名帖后选择同意或拒绝，再点击“保存名帖”。
5. 管理员同意后，用户退出并重新进入问云小院，即可看到正式资料更新。

### 用户后台看不到归云堂二维码弹窗

可能原因：

1. 没有执行 `supabase/migrations/20260501113000_guiyuntang_joined_flag.sql`，普通成员还没有新版读取归云堂二维码的 RLS 权限。
2. 名帖状态还不是“已通过”或“已联系”。
3. 管理员还没有在 `/admin/settings` 上传并保存二维码。
4. 用户已经点击过“我已入群”，或管理员已经勾选“已入归云堂”，所以不会再主动弹窗。

解决方法：

1. 先执行 `supabase/migrations/20260429173000_guiyuntang_qr_settings.sql`。
2. 再执行 `supabase/migrations/20260501110000_unify_roster_profile_fields.sql`。
3. 再执行 `supabase/migrations/20260501113000_guiyuntang_joined_flag.sql`。
4. 后台把成员名帖状态改为“已通过”或“已联系”。
5. 成员刷新 `/yard`，系统会全屏弹出二维码，并提供“我已入群”和“暂时关闭”两个按钮。
6. 如果成员点击过“暂时关闭”，只是本次关闭；如果点击过“我已入群”或管理员已勾选“已入归云堂”，不会再主动弹窗，但可在小院总览里点击“查看入群二维码”或“再次查看二维码”。

### 问云考核结果在小院或后台看不到

原因：没有执行问云考核 SQL，或者当前页面还停留在旧部署版本。

解决方法：

1. 执行 `supabase/migrations/20260429110500_wenxin_quiz_gate.sql`。
2. 重新部署前端网站。
3. 退出账号后重新登录。
4. 再次提交一遍问云考核，确保数据库里有最新记录。

### 手机端导航自动展开又收回

这是正常设计。手机首次打开官网时，顶部导航会自动展开，方便新用户发现菜单入口；两秒后会自动收回，避免遮挡页面内容。

### 需要登录的操作无法继续

原因：名册登记、问云考核提交、云灯留言和雅集报名都需要绑定问云小院账号，未登录时不会直接写入数据。

解决方法：

1. 点击页面里的“去登录或注册”。
2. 在 `/login` 使用邮箱或旧编号登录。
3. 登录成功后回到原页面继续操作。
4. 如果是新用户登记入册，请先完成 `/wenxin-quiz`，再回到 `/join` 提交名帖。

### 执行名册升级脚本时报 member_code 改名失败

原因：旧版 `roster_entries` 公开视图里第三列是 `member_code`，新版需要新增 `jianghu_name`。PostgreSQL 不允许直接用 `create or replace view` 改变已有视图的列顺序，所以会提示无法把 `member_code` 改成 `jianghu_name`。

解决方法：

1. 使用最新的 `supabase/migrations/20260428125300_roster_names_and_review.sql`。
2. 重新复制整段脚本到 Supabase SQL 编辑器执行。
3. 最新脚本会先删除旧的 `roster_entries` 公开视图，再按新字段重建视图。
4. 如果之前执行到一半失败，直接重新执行最新脚本即可，前面的字段和约束语句都是可重复执行的。

### 云灯提交后前台看不到

这是正常设计。云灯默认是 `pending` 待审核，管理员在后台通过后才会公开。

### GitHub Pages 部署后页面资源丢失

本项目已把 Vite 的 `base` 设置为相对路径，一般不会丢资源。

如果仍然出错，请确认：

1. GitHub Pages 使用的是 GitHub Actions 部署。
2. Actions 已成功生成 `dist`。
3. 仓库密钥名称与文档一致。

### 运行构建时出现体积提醒

当前构建会生成完整前台和后台脚本，体积提醒不影响运行。

后续如果访问量变大，可以把后台页面改成按需加载，减少首屏脚本大小。
